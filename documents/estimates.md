**見積作成ツール**

> 本ツールは「OVER Inc. 社内ツール統合プラットフォーム」の一部として、企画書作成システムと同一のNext.jsプロジェクトに統合される。詳細なシステム構成・認証・DB設計は統合版要件定義書（`OVER_統合プラットフォーム_要件定義書_v2.md`）を参照。

## 1. 背景・目的

### 1.1 現状の課題

営業担当者がマネーフォワードで見積書を作成し、PDF形式でクライアントへ提出しているが、以下の問題が発生している。

- **項目の抜け漏れが多く**、後から追加費用が発生し利益を圧迫している
- **工数の見積もりが感覚的**で根拠がなく、精度にばらつきがある
- **原価（内部工数・外注費）が可視化されておらず**、粗利を意識した見積作成が難しい
- **必要な但し書き・免責事項の記載が不十分**で、スコープトラブルのリスクがある
- **過去の見積知見が蓄積されず**、毎回ゼロから作成している

### 1.2 ツール導入の目的

- 案件タイプ別のおすすめ明細（品目・詳細・工数・金額付き）により、抜け漏れを構造的に防ぐ
- **AIアシスタントが過去の見積実績を参照**し、案件に合った明細を対話形式で提案する
- 内部工数・外注費の入力を義務化し、粗利をリアルタイムで可視化する
- 備考・免責事項のテンプレート（43種）を案件タイプに応じて自動挿入し、リスクヘッジを徹底する
- 見積データを蓄積し、AIアシスタントの精度向上と工数削減につなげる
- 予実管理スプレッドシートへの連携データを自動生成する

#### おすすめ明細パネル（社内機能）

選択した案件タイプに応じた定番品目を表示するパネル。各品目には以下の情報が含まれる:

| 項目 | 説明 |
|------|------|
| カテゴリ | 作業カテゴリ（企画・設計、デザイン、コーディング等） |
| 品目名 | 品目の名称 |
| 詳細（description） | 品目に含まれる作業内容の説明（1〜2文） |
| 単価 | 標準単価（カンマ区切りで表示） |
| 数量 | デフォルト数量 |
| 単位 | 式、ページ、本、点、人、日、月、回、時間、部、セット |
| 内部工数（internalDays） | 推定工数（人日単位） |

**3段階の追加操作:**
1. **個別追加**: 品目ごとの「+」ボタンで1件ずつ追加
2. **カテゴリ追加**: カテゴリ単位でまとめて追加
3. **全件追加**: 「全て追加」ボタンで表示中の全品目を一括追加

**重複防止**: 既に明細テーブルに追加済みの品目は ✓ マーク表示、追加ボタンが非活性になる（品目名の完全一致で判定）。

2行構造のテーブル。各明細は1行目がメイン情報、2行目が詳細説明。

```

├──────┴──────┴──────┴────┴────┴────┴──────┴────┴────┴────┴──┤
│ 📝 詳細内容のテキストエリア（品目に含まれる作業内容の説明）   │
└─────────────────────────────────────────────────────────────┘
```

| 項目 | 説明 | PDF出力 |
|------|------|:---:|
| カテゴリ | 作業カテゴリ名 | ○ |
| 品目名 | 品目の名称 | ○ |
| 詳細（description） | 品目に含まれる作業内容（テキストエリア、全幅） | ○ |
| 単価 | 金額入力（カンマ区切り表示・フォーカス時は数値のみ） | ○ |
| 数量 | 数量 | ○ |
| 単位 | セレクト（11種） | ○ |
| 金額 | 単価 × 数量（自動計算・カンマ区切り表示） | ○ |

#### AI見積アシスタント（右カラム）

過去の見積実績を参照して、案件に合った明細を対話形式で提案するチャットUI。

```
┌──────────────────────┐
│ AI見積アシスタント 🤖  │  ← 開閉ボタン
├──────────────────────┤
│ メッセージ表示エリア   │
│ (スクロール可能)       │
│                      │
│ AI: こんにちは！      │
│ どんな案件ですか？    │
│                      │
│ User: Webリニューアル │
│ で15ページくらい...   │
│                      │
│ AI: なるほど、では    │
│ こちらの明細を提案    │
│ します:              │
│ ┌──────────────────┐ │
│ │ 提案された明細    │ │
│ │ (プレビュー表示)  │ │
│ │ [明細に追加する]  │ │
│ └──────────────────┘ │
├──────────────────────┤
│ [メッセージ入力]  [→] │
└──────────────────────┘
```

| 項目 | 仕様 |
|------|------|
| AIモデル | Claude Sonnet 4.5（`claude-sonnet-4-5-20250929`） |
| 通信方式 | Vercel AI SDK ストリーミング（`useChat` + `DefaultChatTransport`） |
| APIエンドポイント | `/api/ai/quote-suggest` |
| 過去実績参照 | チーム内の提出済み・受注済み見積を最大10件参照（案件タイプ部分一致） |
| 会話履歴管理 | `windowByText()` で最大20メッセージ / 16,000文字に制限 |
| 使用量記録 | `ai_usage_logs` テーブルに自動記録 |

**AIの動作フロー:**
1. ユーザーに案件の概要を1〜3回のやりとりで聞き出す
2. 過去の見積実績を参考に単価・工数の妥当性を判断
3. 提案が固まったら、JSONブロック付きで明細を出力
4. UIがJSONを自動検出 → 「明細に追加する」ボタンを表示
5. ボタン押下で `dispatch({ type: "ADD_SUGGESTED_ITEMS" })` を実行

**AIが出力するJSON形式:**
```json
{
  "items": [
    {
      "category": "カテゴリ名",
      "name": "品目名",
      "description": "この品目に含まれる作業内容の説明",
      "unitPrice": 200000,
      "quantity": 1,
      "unit": "式",
      "internalDays": 3
    }
  ]
}
```

#### 粗利ダッシュボード（社内専用・リアルタイム）

| 表示項目 | 計算式 |
|----------|--------|
| 売上合計（税抜） | 明細合計 |
| 内部人件費合計 | 人日 × 人月単価 ÷ 20営業日 |
| 外注費合計 | 各行の外注費見込の合計 |
| 原価合計 | 内部人件費 ＋ 外注費 |
| 粗利 | 売上合計 － 原価合計 |
| 粗利率 | 粗利 ÷ 売上合計 × 100（%表示・色で警告） |

**粗利率の警告表示**

| 粗利率 | 表示 |
|--------|------|
| 50%以上 | 緑（良好） |
| 30〜49% | 黄（要注意） |
| 30%未満 | 赤（要確認） |

## 5. データ仕様・保存

### 5.1 基本仕様

| 項目 | 仕様 |
|------|------|
| 保存場所 | Supabase PostgreSQL（クラウド保存・複数端末対応） |
| 主要テーブル | `quotes`（見積ヘッダー）、`quote_items`（見積明細）、`ai_usage_logs`（AI使用ログ） |
| 見積番号採番ルール | YYYYMMDD-連番（例：20260227-01） |
| 人月単価（固定） | ディレクター：3,000,000円 / 月　営業：2,000,000円 / 月 |
| 1日あたり換算 | 人月単価 ÷ 20営業日（ディレクター：150,000円/日、営業：100,000円/日） |
| 消費税率 | 10%（固定） |
| 端数処理 | 円未満切り捨て |
| アクセス制御 | Supabase RLS（Row Level Security）でチームメンバー以外のアクセスを防止 |

## 7. 非機能要件・スコープ外

### 7.1 非機能要件

- レスポンシブ対応（PC主体、xlブレークポイントで2カラム化）
- 見積の計算はリアルタイムで更新
- Supabase RLSによりチームメンバー以外のデータアクセスを防止
- AI使用量を `ai_usage_logs` テーブルに記録（コスト管理用）

### 7.2 スコープ外（現フェーズ）

- マネーフォワードとのAPI連携
- 承認ワークフロー
- 見積一覧の検索・フィルター機能
- 外注先ごとの発注管理
- 企画書プロジェクトとの直接連携（将来拡張として統合版DBスキーマで準備済み）

## 8. 技術方針

| 項目 | 採用技術・方針 |
|------|--------------|
| フレームワーク | Next.js 16 (App Router / Turbopack)。企画書システムと同一プロジェクト |
| スタイリング | Tailwind CSS 4（`@theme inline` CSS変数ベース） |
| 状態管理 | React `useReducer`（QuoteFormState / QuoteAction） |
| データ永続化 | Supabase PostgreSQL（`quotes`・`quote_items` テーブル） |
| 認証 | Supabase Auth。企画書システムと共通 |
| AI | Claude Sonnet 4.5（`claude-sonnet-4-5-20250929`）。AI見積アシスタントに使用 |
| AI SDK | Vercel AI SDK（`streamText` / `useChat` / `DefaultChatTransport`） |
| ドラッグ&ドロップ | @dnd-kit/core + @dnd-kit/sortable（明細行の並び替え） |
| PDF生成 | ブラウザ印刷（@media print）。品質問題時はjsPDFに切り替え |
| CSV出力 | JavaScript（Blob API）でクライアント生成 |
| デプロイ | Vercel。企画書システムと同一プロジェクトとしてデプロイ |

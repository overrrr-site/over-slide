# OOUI（オブジェクト指向UI）設計ガイドライン

> 本プロジェクトのUI設計で参照するためのコンテキスト文書
> 作成日：2026-03-01

---

## 1. OOUIとは

OOUI（Object-Oriented User Interface）は、**「モノ（名詞）を起点としてUIを設計する考え方」**。

ユーザーが画面上で操作する順序を「**対象を選ぶ → 操作を選ぶ**」に統一することで、現実世界の行動パターンとデジタル操作を一致させる。

### 日常の例

- 買い物：**商品を手に取る**（対象）→ **カゴに入れる**（操作）
- 音楽：**曲を選ぶ**（対象）→ **再生する**（操作）

これをそのままUIに反映する。

---

## 2. 4つの原則

### 原則1：知覚と直接的操作

画面上のオブジェクトが**視覚的に理解でき、直感的に触れる**こと。

- オブジェクトは見た目で何か分かる形にする
- クリック・タップで直接操作できる
- 隠れた操作や記憶に頼る操作を減らす

### 原則2：性質と状態の体現

操作に対する**即時の反応（フィードバック）で状態を伝える**こと。

- 選択中の項目はハイライトで示す
- 保存中・処理中などの状態を視覚的に表現する
- 変更が反映されたことを即座にフィードバックする

### 原則3：オブジェクト → アクション順

ユーザーは**まず対象（名詞）を選び、次に操作（動詞）を選ぶ**。

- 「何を」→「どうする」の順序を一貫させる
- 操作メニューは対象を選んだ後に表示する
- コンテキスト（文脈）に応じた操作だけを提示する

### 原則4：協調的構成

すべてのオブジェクトが**一貫したルールで構成される**こと。

- 同じ種類のオブジェクトは同じ見た目・振る舞いにする
- 重要なものは大きく、補助的なものは控えめに
- 類似したものは近くにまとめ、異なるものは離す

## 3. タスク指向UIとの比較

| 観点 | OOUI（オブジェクト指向） | タスク指向UI |
|------|------------------------|-------------|
| 操作の順序 | 対象を選ぶ → 操作を選ぶ | 操作を選ぶ → 対象を選ぶ |
| 画面の起点 | オブジェクトの一覧 | 機能・操作のメニュー |
| 向いている場面 | 対象が多い・自由に探索する場面 | 目的が明確・手順が決まっている場面 |
| 例 | ECサイトの商品一覧 → 詳細 → 購入 | ATMの「引き出す」→ 金額入力 → 確認 |
| 画面数 | 少なくなる傾向 | 多くなる傾向 |
| ユーザーの自由度 | 高い | 低い（ガイドされる） |

### どちらを使うか

- **OOUI が向く場面**：操作の対象がたくさんある、ユーザーが自由に画面を行き来する
- **タスク指向が向く場面**：目的が1つに決まっている、確実にゴールまで誘導したい

**本プロジェクトでは**：企画書のダッシュボード・一覧画面はOOUI、ウィザード形式の工程進行はタスク指向の要素も残す（ハイブリッド設計）。

## 4. 設計プロセス（3ステップ）

### Step 1：オブジェクト抽出

ユーザーがやりたいことから**名詞を抽出**し、主要なオブジェクトを特定する。

1. ユーザーの行動を書き出す（例：「企画書を作る」「リサーチ結果を確認する」）
2. 行動から**名詞**を抽出する（例：企画書、リサーチ結果、構成、レビュー）
3. 名詞の関係を整理する（親子関係、所有関係など）
4. **メインオブジェクト**を特定する（最も操作頻度が高い・重要なもの）
5. 付随する情報を**プロパティ（属性）**として整理する

### Step 2：ビューとナビゲーション設計

各オブジェクトに**2種類のビュー**を用意する。

| ビュー | 役割 | 例 |
|--------|------|---|
| コレクションビュー（一覧） | 複数のオブジェクトを並べて見せる | プロジェクト一覧、見積一覧 |
| シングルビュー（詳細） | 1つのオブジェクトの全情報を見せる | プロジェクト詳細、見積編集画面 |

**ナビゲーションの原則**：

- 一覧 → 詳細の階層構造を基本とする
- オブジェクト間の関連を辿れる導線を用意する
- 「今どこにいるか」が常に分かるようにする（パンくずリスト等）

### Step 3：レイアウトパターンの適用

オブジェクトの特性に合わせた表示形式を選ぶ。

| パターン | 向いているオブジェクト | 例 |
|----------|---------------------|---|
| カードグリッド | ビジュアル情報を持つもの | プロジェクト一覧 |
| テーブル（表形式） | 定量的情報・比較したいもの | 見積一覧、明細テーブル |
| リスト | テキスト中心の項目 | レビュー結果、ナレッジ一覧 |
| タイムライン | 時系列で並ぶもの | 工程の進行状況 |

## 5. 本プロジェクトへの適用メモ

### メインオブジェクト候補

| ツール | メインオブジェクト | 関連オブジェクト |
|--------|------------------|-----------------|
| 企画書作成 | プロジェクト | ブリーフシート、リサーチメモ、構成、ページ、レビュー |
| 見積作成 | 見積 | 明細、テンプレート、サジェスト品目 |
| 共通 | ナレッジ | ナレッジ文書、スタイルガイド |

### ビュー設計の方向性

```
プロジェクト一覧（コレクション）
  └→ プロジェクト詳細（シングル）
      ├→ ブリーフシート
      ├→ リサーチメモ
      ├→ 構成（ページ一覧）
      │   └→ ページ詳細
      ├→ レビュー結果
      └→ 出力ファイル

見積一覧（コレクション）
  └→ 見積詳細（シングル）
      ├→ 基本情報
      ├→ 明細テーブル
      ├→ 原価パネル
      └→ 粗利ダッシュボード
```

# OOUI + HIG 本プロジェクトへの適用ガイド

> OVER Inc. 社内ツール統合プラットフォームのUI設計指針
> 作成日：2026-03-01

---

## 1. 目的

OOUIの設計手法とHIGのデザイン原則を組み合わせ、本プロジェクト（企画書作成システム＋見積作成ツール）のUIを設計するための具体的な指針を示す。

---

## 2. オブジェクト分析

### Step 1：オブジェクト抽出

要件定義書のユーザー行動から名詞を抽出し、メインオブジェクトを特定する。

#### 企画書・ドキュメント作成システム

| オブジェクト | 種類 | 主なプロパティ | 主なアクション |
|------------|------|--------------|--------------|
| **プロジェクト** | メイン | タイトル、出力タイプ、ステータス、現在工程、作成日 | 作成、開く、削除、進行 |
| ブリーフシート | 子（プロジェクトの） | クライアント情報、背景、仮説、ゴール、制約、トーン | 編集、確定 |
| リサーチメモ | 子（プロジェクトの） | 検索結果、アップロードデータ、メモ内容 | 検索実行、編集、追記 |
| 構成 | 子（プロジェクトの） | ページ一覧、ページメッセージ、バージョン | 編集、並替え、確定 |
| ページ | 子（構成の） | ページ番号、コンテンツ、マスター種別 | 編集、再生成、削除 |
| レビュー | 子（プロジェクトの） | 総評、指摘一覧、改修プラン | 閲覧、指摘の採否切替 |
| 出力ファイル | 子（プロジェクトの） | ファイル形式、バージョン、パス | ダウンロード、プレビュー |

#### 見積作成ツール

| オブジェクト | 種類 | 主なプロパティ | 主なアクション |
|------------|------|--------------|--------------|
| **見積** | メイン | 見積番号、案件名、クライアント、ステータス、合計金額、粗利率 | 作成、編集、複製、PDF出力、CSV出力 |
| 明細行 | 子（見積の） | カテゴリ、品目名、単価、数量、金額、内部工数、外注費 | 追加、編集、削除、並替え |
| サジェスト品目 | 関連 | 品目名、標準単価、対象案件タイプ | 明細に追加 |

#### 共通

| オブジェクト | 種類 | 主なプロパティ | 主なアクション |
|------------|------|--------------|--------------|
| ナレッジ文書 | メイン | タイトル、モード（スタイル/内容）、タグ、解析状態 | アップロード、タグ付け、削除 |
| スタイルガイド | メイン | 構成パターン、文体、情報密度、フレーズ、カスタムルール | 確認、修正、ルール追加 |

## 3. ビュー設計の方針

### 基本構造：コレクション → シングルの階層

OOUIの基本に従い、各メインオブジェクトに**一覧ビュー**と**詳細ビュー**を用意する。

```
┌─ グローバルナビゲーション（フラット型）──────────────────┐
│  ダッシュボード  │  企画書  │  見積  │  管理            │
└──────────────────────────────────────────────────────┘

■ 企画書セクション
  プロジェクト一覧（コレクション / カードグリッド）
    └→ プロジェクト詳細（シングル / 工程ステップバー付き）
        ├→ ディスカッション（チャットUI）
        ├→ リサーチメモ（テキスト編集）
        ├→ 構成（ページ一覧 + メッセージ確定）
        ├→ ページ詳細（コンテンツ編集）
        ├→ プレビュー
        ├→ レビュー（指摘一覧）
        └→ 出力（ダウンロード）

■ 見積セクション
  見積一覧（コレクション / テーブル）
    └→ 見積詳細（シングル / タブ or パネル分割）
        ├→ 基本情報
        ├→ 明細テーブル（+ サジェストパネル）
        ├→ 原価入力パネル
        └→ 粗利ダッシュボード

■ 管理セクション
  ├→ ナレッジベース（コレクション / テーブル）
  │     └→ ナレッジ詳細（シングル）
  ├→ スタイルガイド（シングル / 4カテゴリ表示）
  └→ テンプレート設定（シングル）
```

### レイアウトパターンの使い分け

| 画面 | パターン | 理由 |
|------|---------|------|
| プロジェクト一覧 | カードグリッド | ステータスや出力タイプを視覚的に区別しやすい |
| 見積一覧 | テーブル | 金額・粗利率など数値の比較が主目的 |
| ナレッジ一覧 | テーブル | タグ・モード・解析状態で絞り込む用途 |
| プロジェクト詳細 | ステップバー + 作業パネル | 工程の進行を視覚化しつつ、各工程の操作に集中 |
| 見積詳細 | サイドパネル分割 | 明細とサジェストを同時に見ながら操作 |

## 4. ナビゲーション構造

### グローバルナビゲーション（HIG：フラット型）

ヘッダーに常時表示。どの画面からでもトップレベルに移動できる。

```
[ロゴ] [ダッシュボード] [企画書を作る] [見積を作る] [管理] [ユーザー]
```

### ローカルナビゲーション（HIG：階層型）

各セクション内は一覧 → 詳細の階層で遷移。

- **パンくずリスト**で現在位置を常に表示
- **戻るボタン**で親階層に戻れる
- プロジェクト詳細内は**ステップバー**で工程間を移動

### コンテキストナビゲーション（HIG：コンテンツ駆動型）

関連オブジェクト間の直接遷移。

- レビュー画面の指摘 → 該当ページへ直接遷移
- ブリーフシート → リサーチメモの該当箇所
- ナレッジ参照の出典 → ナレッジ文書の詳細

## 5. HIG原則の具体的な反映

### Clarity（明瞭さ）の適用

| 対象 | 具体策 |
|------|--------|
| プロジェクトステータス | テキスト + 色 + アイコンの3要素で表現（色だけに頼らない） |
| 工程の進行状態 | ステップバーで「完了・進行中・未着手」を視覚的に区別 |
| ボタンラベル | 「保存」「次の工程へ」「レビューを開始」など具体的な動詞 |
| エラー表示 | 「何が問題か」+「どうすれば直るか」をセットで表示 |

### Deference（控えめさ）の適用

| 対象 | 具体策 |
|------|--------|
| プレビュー画面 | UI要素を最小化し、企画書/ドキュメントの内容に集中 |
| ディスカッション画面 | チャット内容が主役、モード切替やトラッカーは控えめに配置 |
| 見積編集画面 | 明細テーブルが主役、サジェストパネルは折りたたみ可能に |
| 全体のカラースキーム | Navy/Beige/Off Whiteの落ち着いた配色でコンテンツを引き立てる |

### Depth（奥行き）の適用

| 対象 | 具体策 |
|------|--------|
| 画面遷移 | 一覧 → 詳細はスライドイン、詳細 → 一覧はスライドアウト |
| モーダル | 背景をぼかし、前面に浮き上がる表現 |
| ステップバー | 完了工程は沈み込み、現在工程は浮き上がり |
| サイドパネル | メインエリアとの間に影を付けて層を表現 |

### Consistency（一貫性）の適用

| 対象 | 具体策 |
|------|--------|
| ボタンスタイル | プライマリ（Navy）、セカンダリ（ボーダー）、危険（赤）を全画面統一 |
| アイコン | Iconifyの同一セットを全画面で使用 |
| 用語 | 「プロジェクト」「見積」「工程」など用語を統一（表記揺れ防止） |
| レイアウト | ヘッダー高さ、サイドバー幅、余白サイズを全画面で統一 |
| 操作パターン | 「作成」は右上、「削除」は最後、「戻る」は左上に固定配置 |

## 6. OOUI + HIG の設計チェックリスト

### 画面設計時に確認する項目

**OOUI観点：**
- [ ] 画面の起点は「オブジェクト」になっているか（操作メニューからではなく）
- [ ] 「対象を選ぶ → 操作を選ぶ」の順序になっているか
- [ ] メインオブジェクトに一覧ビューと詳細ビューがあるか
- [ ] オブジェクトの状態がリアルタイムで視覚に反映されているか
- [ ] 同じ種類のオブジェクトは同じ見た目で統一されているか

**HIG Clarity（明瞭さ）観点：**
- [ ] 文字の階層（見出し・本文・補足）が統一されているか
- [ ] ボタンラベルは「何が起きるか」を明確に書いているか
- [ ] 情報の優先度が視覚的に表現されているか
- [ ] 色だけに意味を持たせていないか（テキストやアイコンでも補完）

**HIG Deference（控えめさ）観点：**
- [ ] ユーザーの作業内容（コンテンツ）が画面の主役になっているか
- [ ] UIの装飾がコンテンツの邪魔をしていないか
- [ ] 不要なモーダルやポップアップを使っていないか

**HIG Depth（奥行き）観点：**
- [ ] ユーザーが「今どこにいるか」を常に把握できるか
- [ ] 画面遷移の方向が階層構造と一致しているか
- [ ] モーダルやパネルの重なりが自然に見えるか

**HIG Consistency（一貫性）観点：**
- [ ] 同じ操作が同じ場所・同じ見た目で提供されているか
- [ ] 用語やラベルが全画面で統一されているか
- [ ] コンポーネントのスタイルがブランドガイドに沿っているか

## 7. タスク指向UIとの使い分け（ハイブリッド設計）

本プロジェクトでは、画面の性質に応じてOOUIとタスク指向UIを使い分ける。

| 画面 | 採用するUI思想 | 理由 |
|------|---------------|------|
| ダッシュボード | OOUI | プロジェクト・見積のオブジェクト一覧が起点 |
| プロジェクト一覧 | OOUI | 複数のプロジェクトから自由に選択 |
| 見積一覧 | OOUI | 複数の見積から自由に選択・比較 |
| 工程進行（ステップバー） | タスク指向 | 決まった順序でゴールへ誘導する必要がある |
| ディスカッション | タスク指向 | 対話という行為自体がタスク |
| 見積PDF出力 | タスク指向 | 出力という明確なゴールがある |
| ナレッジ管理 | OOUI | ナレッジ文書のオブジェクト一覧が起点 |
| スタイルガイド | OOUI | スタイルガイドという1つのオブジェクトの詳細表示 |

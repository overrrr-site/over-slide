# OOUI + HIG 全面リデザイン計画

> OVER Inc. 社内ツール統合プラットフォーム（OVERworks）のUI改善計画
> 作成日：2026-03-01
> ステータス：承認済み

---

## 1. 背景と目的

### なぜ改善するか

現在のUIは機能としては動いているが、以下の課題がある。

| 観点 | 現状の課題 |
|------|----------|
| OOUI：オブジェクト起点 | ダッシュボードが「プロジェクト一覧」のみ。見積・ナレッジが別画面に分断されており、全体像が掴みにくい |
| OOUI：一覧→詳細 | プロジェクト一覧がテキスト主体のリストで、進捗や状態が一目で分かりにくい |
| HIG：Clarity | 工程の進捗がテキスト表示のみ。視覚的な進捗表現が弱い |
| HIG：Deference | サイドバーとヘッダーが目立ち、作業エリアの面積が狭く感じる |
| HIG：Depth | パンくずリストがなく、プロジェクト詳細画面で「今どこにいるか」が分かりにくい |
| HIG：Consistency | UIコンポーネントがページ内に直書きされており、ボタン・カード等の見た目にばらつきがある |

### 目的

OOUI（オブジェクト指向UI）の設計手法とHIG（ヒューマンインターフェースガイドライン）のデザイン原則を適用し、**「どこに何があるかすぐ分かる」「見た目が統一されたプロ品質の」UI**に刷新する。

### 参照ドキュメント

- `documents/ooui-guidelines.md` — OOUI設計ガイドライン
- `documents/hig-design-principles.md` — HIGデザイン原則ガイド
- `documents/ooui-hig-application.md` — 本プロジェクトへの適用ガイド
- `documents/requirements.md` — 要件定義書

## 2. オブジェクトモデル

### メインオブジェクト（一覧画面を持つもの）

| オブジェクト | 説明 | 一覧ビュー | 詳細ビュー |

### 子オブジェクト（メインの中に含まれるもの）

| 子オブジェクト | 親 | 表示場所 |
|--------------|---|---------|
| ブリーフシート | プロジェクト | ディスカッション画面のサイドパネル |
| リサーチメモ | プロジェクト | リサーチ工程の作業エリア |
| 構成・ページ | プロジェクト | 構成工程のページ一覧 + 各ページ詳細 |
| レビュー | プロジェクト | レビュー工程の指摘一覧 |
| 明細行 | 見積 | 見積詳細のテーブル行 |

### 設定オブジェクト（単一インスタンス）

| オブジェクト | 説明 | ビュー |
|------------|------|--------|
| スタイルガイド | 文体・構成パターンの設定 | 詳細ビューのみ（4カテゴリ編集） |
| テンプレート設定 | カラースキーム・フォント | 詳細ビューのみ（設定画面） |

### ナビゲーション設計（3層）

#### グローバルナビ（サイドバー）— フラット型
- どの画面からでもトップレベルに移動可能
- サイドバーは常時表示（lg以上）、モバイルはハンバーガー

#### ローカルナビ（パンくず + 戻る）— 階層型
- 全画面にパンくずリストを配置
- 例：`ホーム > プロジェクト > ABC提案書 > 構成作成`
- プロジェクト詳細では「← プロジェクト一覧」の戻り導線

#### コンテキストナビ（コンテンツ内リンク）— コンテンツ駆動型
- レビュー指摘 → 該当ページへ直接遷移
- ブリーフシート → リサーチメモの該当箇所

### 画面一覧と遷移

```
ホーム（/dashboard）
  ├→ プロジェクト一覧（/dashboard?tab=projects）
  │   └→ プロジェクト詳細（/projects/[id]）
  │       ├→ ディスカッション（/projects/[id]/discussion）
  │       ├→ リサーチ（/projects/[id]/research）
  │       ├→ 構成（/projects/[id]/structure）

  │       ├→ デザイン（/projects/[id]/design）
  │       └→ デザインレビュー（/projects/[id]/design-review）
  ├→ 見積一覧（/quotes）
  │   └→ 見積詳細（/quotes/[id]）
  ├→ ナレッジ（/knowledge）
  ├→ スタイルガイド（/settings/style-guide）
```

## 4. 各画面の改善設計

### 4.1 ダッシュボード（ホーム）の統合ハブ化

**現状**：プロジェクト一覧のみ
**改善後**：全オブジェクトの入り口 + クイックアクション

```
┌──────────────────────────────────────────────────┐
│  ホーム                                           │
│                                                  │
│  ■ 最近のプロジェクト                               │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐         │
│  │ PJ名      │ │ PJ名      │ │ PJ名      │        │
│  │ クライアント │ │ クライアント │ │ クライアント │        │
│  │ ■■■■□□□   │ │ ■■■■■■□   │ │ ■■□□□□□   │        │
│  │ 構成作成    │ │ レビュー    │ │ リサーチ    │        │
│  │ 進行中     │ │ レビュー    │ │ 進行中     │        │
│  └──────────┘ └──────────┘ └──────────┘         │
│                          [すべてのプロジェクト →]    │
│                                                  │
│  ■ 最近の見積                                      │
│  ┌────────────────────────────────────────┐      │
│  │ 案件名    │ クライアント │ 合計金額   │ 粗利率 │ 状態  │
│  │──────────────────────────────────────│      │
│  │ Web制作   │ ABC株式会社 │ ¥1,500,000│ 🟢52% │ 提出済│
│  │ 動画制作  │ DEF商事    │ ¥800,000 │ 🟡35% │ 下書き│
│  └────────────────────────────────────────┘      │
│                          [すべての見積 →]          │
│                                                  │
│  ■ クイックアクション                               │
│  [+ 新規プロジェクト]  [+ 新規見積]                  │
│                                                  │
└──────────────────────────────────────────────────┘
```

**OOUI観点**：メインオブジェクト（プロジェクト＋見積）を1画面で把握 → オブジェクト起点の体験
**HIG観点**：Clarity（情報の優先度が一目で分かる）、Deference（コンテンツが主役）

### 4.2 プロジェクト一覧の刷新

**現状**：テキスト主体のリスト表示
**改善後**：カードグリッド + 進捗バー + フィルター

**プロジェクトカード**：
```
┌────────────────────────────────────┐
│  📄 ABC株式会社 提案書               │  ← タイトル（太字）
│  ABC株式会社                        │  ← クライアント名
│                                    │
│  ■■■■■□□  5/7  構成作成            │  ← 進捗バー + 現在工程
│                                    │
│  🎬 スライド  │  進行中  │  3/1更新   │  ← 出力タイプ + 状態 + 日付
└────────────────────────────────────┘
```

**フィルター**：
- ステータス別（全て / 進行中 / レビュー / 完了 / 下書き）
- 出力タイプ別（全て / スライド / ドキュメント）

### 4.3 見積一覧のテーブル化

**現状**：カード形式のリスト
**改善後**：テーブル形式（数値の比較に最適化）

| 見積番号 | 案件名 | クライアント | 合計金額（税込） | 粗利率 | 状態 | 更新日 | 操作 |
|---------|--------|------------|---------------|--------|------|--------|------|
| 20260301-001 | Webサイト制作 | ABC株式会社 | ¥1,650,000 | 🟢 52% | 提出済み | 3/1 | ... |

**OOUI観点**：見積は「数値を見比べる」用途 → テーブルが最適なレイアウトパターン

### 4.4 プロジェクト詳細ヘッダーの強化

**現状**：タイトル + クライアント名 + ステップバー
**改善後**：パンくず + 戻る導線 + 出力タイプ表示 + 改良ステップバー

```
← プロジェクト一覧    ホーム > プロジェクト > ABC提案書
─────────────────────────────────────────────────
ABC提案書                              📄 スライド出力
ABC株式会社

[0.ディスカッション] [1.リサーチ] [2.構成] [3.詳細] [4.デザイン] [5.レビュー] [6.修正]
      ✓                ✓          ●        ○         ○          ○         ○
```

**ステップバーの状態表示**：
- ✓ 完了：Green背景 + チェックマーク
- ● 進行中：Navy背景 + 白文字（現在地を強調）
- ○ 未着手：Off White背景 + グレー文字

**HIG観点**：Depth（パンくずで階層位置を表示）、Clarity（ステップの状態を3段階で区別）

### 4.5 ナレッジ一覧の改善

**現状**：カード形式 + ドラッグ&ドロップアップロード
**改善後**：テーブル形式 + フィルター + ドラッグ&ドロップは維持

- テーブルカラム：タイトル、モード（スタイル学習/内容参照）、タグ、解析状態、更新日
- フィルター：モード別、タグ別、解析状態別

## 5. 共通コンポーネント設計

### 作成するコンポーネント

| コンポーネント | ファイルパス | 用途 |
|--------------|-----------|------|
| Button | `components/ui/button.tsx` | 全操作ボタン（primary / secondary / danger / ghost） |
| Card | `components/ui/card.tsx` | オブジェクト表示枠（default / interactive / selected） |
| Badge | `components/ui/badge.tsx` | ステータス・タグ（draft / active / warning / success / error） |
| DataTable | `components/ui/data-table.tsx` | 見積・ナレッジの一覧テーブル（ソート・フィルター） |
| Breadcrumb | `components/ui/breadcrumb.tsx` | 階層位置表示（全画面共通） |
| PageHeader | `components/ui/page-header.tsx` | ページタイトル + 説明 + アクションボタン |
| ProgressBar | `components/ui/progress-bar.tsx` | 工程進捗表示（プロジェクトカード用） |
| EmptyState | `components/ui/empty-state.tsx` | データなし表示（アイコン + メッセージ + ボタン） |
| SidePanel | `components/ui/side-panel.tsx` | 折りたたみ可能なサイドパネル |
| StepBar | `components/ui/step-bar.tsx` | 工程ナビ（既存を改良、3状態表示） |

### デザイントークン

| トークン | 値 |
|---------|---|
| 間隔 | 4px刻み：4, 8, 12, 16, 24, 32, 48, 64 |
| 角丸 | sm=4px, md=8px, lg=12px, xl=16px |
| 影 | sm（カード）, md（ホバー浮き上がり）, lg（モーダル） |
| フォントサイズ | xs=12px, sm=14px, base=16px, lg=18px, xl=20px, 2xl=24px |

### HIG原則の組み込み

| 原則 | 実装方法 |
|------|---------|
| Clarity | Badgeは必ず色 + テキスト。Buttonラベルは「何が起きるか」を明記 |
| Deference | Card枠線は`border-beige`（控えめ）。作業エリアを最大化 |
| Depth | interactiveカードはホバーで影が深くなる。モーダルは`backdrop-blur` |
| Consistency | 全画面で同じButton/Card/Badgeを使用。配置ルールを統一 |

## 8. OOUI + HIG チェックリスト

### 画面設計時に確認する項目

**OOUI観点：**
- [ ] 画面の起点は「オブジェクト」になっているか
- [ ] 「対象を選ぶ → 操作を選ぶ」の順序になっているか
- [ ] メインオブジェクトに一覧ビューと詳細ビューがあるか
- [ ] オブジェクトの状態がリアルタイムで視覚に反映されているか
- [ ] 同じ種類のオブジェクトは同じ見た目で統一されているか

**HIG Clarity（明瞭さ）：**
- [ ] 文字の階層（見出し・本文・補足）が統一されているか
- [ ] ボタンラベルは「何が起きるか」を明確に書いているか
- [ ] 色だけに意味を持たせていないか

**HIG Deference（控えめさ）：**
- [ ] ユーザーの作業内容が画面の主役になっているか
- [ ] UIの装飾がコンテンツの邪魔をしていないか

**HIG Depth（奥行き）：**
- [ ] 「今どこにいるか」を常に把握できるか
- [ ] 画面遷移の方向が階層構造と一致しているか

**HIG Consistency（一貫性）：**
- [ ] 同じ操作が同じ場所・同じ見た目で提供されているか
- [ ] 用語やラベルが全画面で統一されているか
